Dataset Iris: Contiene medidas de pétalos y sépalos de tres especies de flores Iris: Setosa, Versicolor y Virginica. Es un dataset perfectamente balanceado y muy utilizado para aprender clasificación.

⚙️ Proceso del Algoritmo Random Forest

1
Preparación de datos: Se elimina la columna ID innecesaria y se identifica la variable objetivo (species).
2
Codificación: Las etiquetas de texto se convierten a números (Setosa=0, Versicolor=1, Virginica=2).
3
Escalado: Se normalizan las características usando StandardScaler para que todas tengan la misma escala.
4
Entrenamiento: Random Forest crea 120 árboles de decisión con profundidad máxima de 4 niveles.
5
Validación: Se usa validación cruzada estratificada de 5 pliegues para evaluar el modelo de forma robusta.

¿Qué significan?

Accuracy: Porcentaje de predicciones correctas del total.

Precision: De las flores que predice como Setosa, ¿cuántas realmente lo son?

Recall: De todas las flores Setosa reales, ¿cuántas logra identificar?

F1-Score: Balance entre precision y recall.

🏆 ¿Qué tan bueno es este modelo?

✅ Fortalezas

  • Accuracy superior al 95% - Excelente

  • Robusto y confiable

⚠️ Consideraciones

Veredicto: Este es un modelo EXCELENTE para el problema planteado. Random Forest es ideal para este tipo de clasificación y los resultados demuestran que puede distinguir muy bien entre las tres especies de Iris basándose en las medidas de pétalos y sépalos.

🧠 ¿Por qué funciona tan bien?

Random Forest

Combina 120 árboles de decisión diferentes. Cada árbol "vota" por una clase, y la mayoría gana. Esto reduce errores y aumenta la confiabilidad.

Dataset Iris

Es un problema "fácil": las tres especies tienen características muy distintivas, especialmente en las medidas de los pétalos.

Validación Cruzada

Divide los datos en 5 partes, entrena en 4 y prueba en 1. Repite 5 veces. Esto da una evaluación muy robusta del modelo.

Escalado de datos

Normaliza todas las medidas para que estén en la misma escala, evitando que una característica domine por ser más grande.
